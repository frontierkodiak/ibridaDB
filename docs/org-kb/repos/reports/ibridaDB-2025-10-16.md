**Purpose & Scope**
- ibridaDB is a modular, reproducible PostgreSQL/PostGIS biodiversity database that: (1) ingests iNaturalist dumps; (2) can enrich observations with MERIT DEM elevation; and (3) exports curated, clade/region-filtered datasets for ML. See overview and stage breakdown. (docs/README.md)
- Core pillars: reproducible ingest; optional elevation via MERIT DEM; curated exports with region bounding boxes, clade filters, and per‑species sampling. (docs/README.md, docs/export.md)

**Versioning Model**
- Two axes: Version v# (schema) vs Release r# (data). Example tags: v0r0 (June 2024), v0r1 (Dec 2024; adds anomaly_score). Recorded in docs and dbTools guide; wrappers encode `VERSION_VALUE`/`RELEASE_VALUE`. (docs/README.md, dbTools/README.md, dbTools/ingest/v0/common/main.sh)
- DB name convention: `ibrida-${VERSION_VALUE}-${RELEASE_VALUE}` (e.g., ibrida-v0-r1) set in wrappers. (dbTools/export/v0/r1/wrapper_*.sh)

**Architecture Overview**
- Stages: Ingestion → Export → Dockerized deployment. (docs/README.md)
- Ingestion: CSV load, geometry (geom.sh), origin/version/release stamping (vers_origin.sh), optional elevation integration. (dbTools/ingest/v0/common/*.sh)
- Export: regional base tables (regional_base.sh), cladistic filtering + CSV (cladistic.sh), unified summary. (dbTools/export/v0/common/*.sh, docs/export.md)
- Docker: custom PostGIS image extends official to include raster2pgsql for DEM loading. Compose variants for local runs. (docker/Dockerfile, docker/stausee/docker-compose.yml)

- Release wrappers: dbTools/ingest/v0/r0|r1/wrapper.sh and export wrappers under dbTools/export/v0/r0|r1/*.sh. (dbTools/ingest/v0/r1/wrapper.sh, dbTools/export/v0/r1/*.sh)

**Export Reference (docs/export.md)**
- Introduction & Pipeline: wrapper → common/main.sh → regional_base.sh → cladistic.sh → CSV + summary; elevation column optional and ordered after longitude. (docs/export.md)
- Environment variables:

**Elevation Data**
- MERIT DEM: GeoTIFF tiles (tar) → PostGIS raster table elevation_raster (rid, rast, filename); ~155 GB, ~2.3M tiles (100×100; WGS84). GIST index on ST_ConvexHull(rast). (docs/elevation_data.md, dbTools/ingest/v0/utils/elevation/create_elevation_table.sql)
- Ingestion commands: download (dbTools/dem/download_merit.sh|download_merit_parallel.sh), load (utils/elevation/load_dem.sh via raster2pgsql), update observations (utils/elevation/update_elevation.sh, parallel). (dbTools/dem/*.sh, dbTools/ingest/v0/utils/elevation/*.sh)
- Observation elevation materialization: adds observations.elevation_meters and fills via spatial join; included in exports when INCLUDE_ELEVATION_EXPORT=true and release != r0. (dbTools/ingest/v0/utils/elevation/update_elevation.sh, dbTools/export/v0/common/functions.sh)

**Indexes for Taxonomy Search & Typus Integration**
- Master index reference: functional, text pattern, trigram, and compound indexes for expanded_taxa. Redundant index to drop: idx_expanded_taxa_taxonid (dup of PK). (docs/expanded_taxa_indexes.md)
- Measured impact: Typus TaxonomyService searches reduced from timeout (>30s) to <1ms after adding pg_trgm + GIN trigram/pattern indexes (~120,000× speedup). (CHANGELOG.md, dev/issues/P0/TYPUS_SEARCH_FIX.md)
- Typus helper usage for consistency:
  - CLI: uv run typus-pg-ensure-indexes --dsn "$POSTGRES_DSN" [--ensure-trgm]. (docs/expanded_taxa_indexes.md)
  - API: ensure_expanded_taxa_indexes(dsn, include_trigram_indexes=True, ...). (docs/expanded_taxa_indexes.md)

**Parameters & Flags (Common)**
- Region/bbox: REGION_TAG resolves to bbox via region_defns.sh; sets XMIN/YMIN/XMAX/YMAX. (dbTools/export/v0/common/region_defns.sh)
- Species threshold: MIN_OBS (typical examples use 50). (dbTools/export/v0/r1/wrapper_*.sh)
- Sampling: MAX_RN caps research‑grade per species via partitioned RN; PRIMARY_ONLY restricts photos to position=0. (dbTools/export/v0/common/cladistic.sh)
- Out‑of‑region handling: INCLUDE_OUT_OF_REGION_OBS toggles inclusion of global obs for in‑region species; in_region boolean computed for each obs. (docs/export.md, dbTools/export/v0/common/regional_base.sh)
- Partial‑rank wipe: MIN_OCCURRENCES_PER_RANK + INCLUDE_MINOR_RANKS_IN_ANCESTORS to drop sparse L20/L30/L40 labels. (dbTools/export/v0/common/cladistic.sh)
- Elevation toggle: INCLUDE_ELEVATION_EXPORT (default true for new releases) inserts elevation_meters right after longitude. (docs/export.md, dbTools/export/v0/common/functions.sh)
- Concurrency knobs: ingest/elevation scripts accept NUM_PROCESSES and parallelize updates by ctid/batch. (dbTools/ingest/v0/common/geom.sh, vers_origin.sh, utils/elevation/update_elevation.sh)
- Safety checks: set -euo pipefail, existence checks, table verification, and summary logging in orchestrators. (dbTools/ingest/v0/utils/elevation/update_elevation.sh, dbTools/ingest/v0/common/vers_origin.sh, dbTools/export/v0/common/main.sh)

**Integration Notes**
- Typus:
  - Relies on expanded_taxa index set for fast search; use Typus index helper to ensure required indexes. (docs/expanded_taxa_indexes.md)
  - ElevationService (draft) expects elevation_raster and queries PostGIS rasters; lon/lat order; returns None for ocean/no‑data. (docs/elevation_data.md)
- Consumers (e.g., Linnaeus):
  - Export CSVs live under ${HOST_EXPORT_BASE_PATH}/${EXPORT_SUBDIR} (e.g., /datasets/ibrida-data/exports/v0/r1/...). (dbTools/export/v0/r1/wrapper_*.sh)
  - Expected columns and order documented, including conditional elevation_meters and anomaly_score. (docs/schemas.md, dbTools/export/v0/common/cladistic.sh)
  - Region and clade filters driven by REGION_TAG and CLADE|METACLADE|MACROCLADE; see clade/region helper files. (dbTools/export/v0/common/clade_defns.sh, clade_helpers.sh, region_defns.sh)
